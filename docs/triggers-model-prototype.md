# Модель триггеров для прототипа

**Статус:** Черновик для прототипа. На созвоне будет подсвечено, что обсуждается уход от этой концепции в сторону работы только с категориями интересов (после дебрифа Tele2 от 27.04.2026).
**Дата:** 29.04.2026

---

## Архитектура

Вертикаль → Интересы → Триггеры. Двухуровневая иерархия.

**Триггер = тип поведения, привязанный к группе доменов.** Например, «посещение сайтов автодилеров» — за этим стоит ведомый Afina список конкретных доменов автодилеров.

## Логика выбора

- Пользователь видит название триггера, без раскрытия списка доменов внутри.
- Можно выбрать один или несколько триггеров в одном сигнале.
- Выбранные триггеры работают по **OR** — срабатывание любого даёт сигнал.
- Триггер можно не донастраивать — он работает с дефолтным списком доменов.
- Донастройка (через AI-fill / промпт-бар): добавить свои домены, исключить свои собственные.

## Ограничения прототипа

- Пользователю доступны только интересы своей вертикали (определяется по анкете).
- Кросс-вертикальный выбор интересов — не сейчас.
- Поиск, иерархия, группировка триггеров — не нужны, список короткий.

## Что не обсуждается на этом этапе

- Временные настройки (частота, период, окно). Это сигналы на основе поведенческих признаков, не рекламные кампании.
- UX просмотра списка доменов внутри триггера. Намеренно закрыто.
