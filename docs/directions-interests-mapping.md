## Назначение

Карта используется для предзаполнения интересов в момент создания первого сигнала. Когда пользователь указал в анкете направление компании, система автоматически предлагает (или предзаполняет галочками) релевантные интересы из общей библиотеки.

## Логика матчинга (модель «намерение клиента»)

Направление = чем занимается компания пользователя.
Интересы = что ищут потенциальные клиенты пользователя в момент готовности к покупке.

Каждому направлению сопоставлены два типа интересов:
- **Прямые** — клиент ищет ровно то, что продаёт пользователь.
- **Смежные** — клиент находится в моменте, после которого нужен продукт пользователя.

Интересы могут принадлежать разным вертикалям. Например, банку релевантны не только финансовые интересы, но и «Покупка квартиры» из вертикали «Недвижимость» (как триггер ипотеки).

## Как это используется в UX

1. Пользователь проходит анкету и указывает направление компании.
2. На фоне работает парсинг сайта (если указан) — он может уточнить, какие именно подкатегории интересов внутри релевантного направления приоритетнее.
3. При создании первого сигнала вертикаль определена (направлением из анкеты), и пользователю показывается список интересов согласно этой карте.
4. Все релевантные интересы могут быть предзаполнены галочками либо показаны без галочек — это решается отдельно при проектировании UX (см. модель триггеров).
