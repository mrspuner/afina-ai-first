# UX настройки триггеров через AI — для прототипа

**Статус:** Готово к передаче в дизайн.
**Дата:** 29.04.2026 (обновлено: триггер раскрывается в карточку, кнопка «Настроить» перемещена внутрь раскрытой карточки)
**Связанные документы:** triggers-model-prototype.md

---

## Главный принцип

Пользователь работает с триггерами через промпт-бар. Дефолтный набор доменов внутри триггера доступен для просмотра — пользователь может раскрыть карточку и увидеть, какие сайты входят в триггер. Пользовательские правки (добавления и исключения) видны как дельта.

## Свёрнутая карточка триггера

В свёрнутом состоянии карточка показывает:

- Чекбокс выбора триггера.
- Название триггера.
- Стрелка раскрытия (⌄) справа.

Если у триггера есть пользовательские правки, под названием отображается блок дельты в свёрнутом виде:

```
[ ] Сайты автодилеров                                    ⌄
```

```
[✓] Сайты автодилеров                                    ⌄
    Добавлено: competitor1.ru, competitor2.ru
    Исключено: mysite.ru
```

## Раскрытая карточка триггера

При клике на стрелку карточка раскрывается и показывает:

- Список доменов, входящих в триггер (дефолтный набор Afina).
- Блок дельты, если есть пользовательские правки (добавленные и исключённые домены).
- Кнопка «Настроить» с иконкой-маскотом (Колобок) — внизу раскрытой карточки.

```
[✓] Сайты автодилеров                                    ⌃
    
    rolf.ru
    avilon.ru
    autoworld.ru
    inchcape.ru
    maxmotors.ru
    
    Добавлено: competitor1.ru, competitor2.ru
    Исключено: mysite.ru
    
    [⚙ Настроить]
```

## Запуск настройки

- Кнопка «Настроить» доступна **только при раскрытой карточке** и **только если триггер выбран чекбоксом**.
- Если триггер не выбран — карточка раскрывается, но кнопка «Настроить» скрыта или disabled.
- Нажатие переносит фокус в промпт-бар.

## Контекст в промпт-баре

- Над промпт-баром появляется подсказка с указанием, какой триггер редактируется. Формат: «Редактируем триггер «X». Напишите, какие сайты добавить или исключить.»
- Подсказка обновляется при переключении на другой триггер.

## Поведение AI на прототипе

- Работа через простой regex: распознаются явно перечисленные домены и команды «добавь / исключи».
- На любой непонятный запрос — единое сообщение: «AI пока не подключён. Сейчас работают команды: «добавь domain1.ru, domain2.ru», «исключи domain3.ru».»
- Сложные кейсы (естественный язык, автоподбор по контексту) — отложены на проработку после прототипа.

## Применение правок

- В момент обработки запроса:
    - У промпт-бара — маленький индикатор обработки.
    - Карточка триггера — лёгкий хайлайт, чтобы взгляд пользователя шёл к месту изменения.
- После применения хайлайт затухает, в карточке появляется или обновляется блок дельты.

## Удаление правок

- Каждая правка отображается чипом с крестиком.
- Клик на крестик — мгновенное удаление, без подтверждения.
- Промпт-бар остаётся способом массовых операций (например, «убери всё, что я добавлял»).

## Переключение между триггерами

- Если в промпт-баре есть введённый, но не отправленный текст — он сохраняется при переключении на другой триггер.
- Достаточно хранить последний черновик; полная история не нужна.
- Подсказка над промпт-баром обновляется на новый контекст.
