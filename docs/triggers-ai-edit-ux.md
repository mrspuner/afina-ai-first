# UX настройки триггеров через AI — для прототипа

**Статус:** Готово к передаче в дизайн.
**Дата:** 29.04.2026
**Связанные документы:** triggers-model-prototype.md

---

## Главный принцип

Пользователь работает с триггерами через промпт-бар. Кнопка «Настроить» у триггера переносит фокус в промпт-бар, и дальнейшее редактирование происходит там. Пользователь не видит дефолтный список доменов внутри триггера — он видит только свои правки (дельту).

## Запуск настройки

- У каждого выбранного триггера есть кнопка «Настроить» с иконкой-маскотом (Колобок).
- Кнопка доступна **только если триггер выбран чекбоксом**. У невыбранных триггеров — скрыта.
- Нажатие переносит фокус в промпт-бар.

## Контекст в промпт-баре

- Над промпт-баром появляется подсказка с указанием, какой триггер редактируется. Формат: «Редактируем триггер «X». Напишите, какие сайты добавить или исключить.»
- Подсказка обновляется при переключении на другой триггер.

## Поведение AI на прототипе

- Работа через простой regex: распознаются явно перечисленные домены и команды «добавь / исключи».
- На любой непонятный запрос — единое сообщение: «AI пока не подключён. Сейчас работают команды: «добавь domain1.ru, domain2.ru», «исключи domain3.ru».»
- Сложные кейсы (естественный язык, автоподбор по контексту) — отложены на проработку после прототипа.

## Применение правок

- В момент обработки запроса:
    - У промпт-бара — маленький индикатор обработки.
    - Карточка триггера — лёгкий хайлайт, чтобы взгляд пользователя шёл к месту изменения.
- После применения хайлайт затухает, в карточке появляется блок дельты.

## Отображение дельты

- Дефолтные домены триггера остаются скрытыми.
- Пользователю показываются только его правки — добавления и исключения.
- Формат отображения:

```
[✓] Сайты автодилеров                    [⚙ Настроить]
    Добавлено: competitor1.ru, competitor2.ru
    Исключено: mysite.ru
```

- Если правок нет — никакого дополнительного блока, чистая карточка.

## Удаление правок

- Каждая правка отображается чипом с крестиком.
- Клик на крестик — мгновенное удаление, без подтверждения.
- Промпт-бар остаётся способом массовых операций (например, «убери всё, что я добавлял»).

## Переключение между триггерами

- Если в промпт-баре есть введённый, но не отправленный текст — он **сохраняется** при переключении на другой триггер.
- Достаточно хранить последний черновик; полная история не нужна.
- Подсказка над промпт-баром обновляется на новый контекст.
