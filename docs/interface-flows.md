# Основные флоу интерфейса

Всё, что пользователь делает в Afina, сводится к пяти функциональным маршрутам. Каждый ведёт от одного понятного действия к одному понятному результату.

---

## Флоу 1. Вход и ориентация

**Стартовый экран** — логотип Afina по центру и слева боковое меню с четырьмя пунктами: **Запустить**, **Статистика**, **Сигналы**, **Кампании**.

**Боковое меню** — главная точка навигации. Нажатие на логотип в любой момент возвращает на стартовый экран. Браузерные кнопки «назад» и «вперёд» работают — можно вернуться к любому предыдущему состоянию.

**«Запустить»** — выдвижная панель слева. Делит на две секции: шаблоны новых сигналов (шесть типовых ситуаций клиентского пути) и живой список уже созданных сигналов, по каждому из которых можно сразу запустить кампанию. Внутри панели есть поиск по обеим секциям.

---

## Флоу 2. Получение первого сигнала

У пользователя есть два способа превратить базу контактов в сигнал.

### Способ А — пошаговый мастер

Из «Запустить» выбирается один из шести шаблонов (Регистрация / Первая сделка / Апсейл / Реактивация / Возврат / Удержание). Открывается восьмишаговый мастер:

1. Подтверждение сценария.
2. Интересы аудитории.
3. Разбиение на четыре ценностных сегмента.
4. Лимиты на количество касаний.
5. Загрузка файла контактов — с анимацией обработки.
6. Сводка введённых данных.
7. Экран обработки с прогресс-баром.
8. Результат: «Сигнал готов» + кнопка перехода к созданию кампании.

Каждый шаг появляется с анимацией — заголовок печатается по буквам, затем проявляется содержимое. Переход вперёд блокируется, пока текущий шаг не заполнен.

### Способ Б — прямая загрузка файла

В разделе **Сигналы** кнопка «Новый сигнал» открывает меню из двух опций. Вариант «Загрузить с устройства» показывает минимальную модалку с drop-zone: CSV, XLSX или TXT до 50 МБ. После выбора файла — короткая анимация хеширования, и сигнал появляется в списке. Без мастера, без настройки сегментов.

**Результат обоих способов:** в разделе «Сигналы» появляется новая карточка.

## Флоу 3. От сигнала к кампании

Кампания всегда привязана к сигналу. Перейти от одного к другому можно тремя путями.

**Путь 1 — с карточки сигнала.** В разделе «Сигналы» у каждой карточки есть кнопка «Создать кампанию». Нажатие сразу открывает canvas с черновиком кампании, названным по типу сигнала («Регистрация #3»). Сценарий в canvas уже собран из шаблона под тип сигнала.

**Путь 2 — из «Запустить».** Та же секция «Новая коммуникационная кампания» в выдвижной панели. Выбор сигнала — мгновенный переход в canvas.

**Путь 3 — сразу после мастера.** Восьмой шаг мастера завершается кнопкой «Запустить кампанию» → экран выбора типа кампании (пять карточек: Возврат брошенных, Прогрев, Стимулирование, Поведенческое, Адаптивное) → canvas.

Во всех случаях пользователь попадает в одно и то же рабочее пространство — canvas кампании.

## Флоу 4. Работа в canvas кампании

Это основная рабочая поверхность. Всё настраивается здесь.

**Шапка кампании.** Название кампании редактируется на месте: клик → ввод → Enter сохраняет, Esc отменяет. Под названием — плашка привязанного сигнала (тип, количество контактов, дата). Если сигнал не привязан — красная плашка с предупреждением. Справа две кнопки: «Сохранить черновик» и «Запустить».

**Сам сценарий.** Граф нод связан стрелками: входной сигнал → шаги коммуникации и логики → итоговая нода «Успех» (или «Конец»). Доступные типы нод:

- **Коммуникация:** SMS, Email, Push, IVR-звонок.
- **Логика:** разветвление, задержка, проверка условия, слияние веток.
- **Веб:** лендинг, витрина.

**Два способа редактировать сценарий.**

*Через клик по ноде.* Нода подсвечивается, над нижней панелью ввода появляется блок с параметрами: тип, имя, подсказки по быстрым действиям (для SMS — «Текст», «Alpha-name», «Ссылка», «Время» и т. д.). Нода-кандидат на настройку становится «адресуемой» — её имя попадает в поле ввода как `@Имя_ноды`.

*Через нижнюю панель ввода (промпт-бар).* Работает в двух режимах:

- **Правка параметров ноды:** `@SMS текст: привет, у нас скидка 15%`.
- **Структурные команды:** `добавь email перед push`, `убери sms`, `замени push на ivr`, `добавь паузу на 2 дня перед email`.

После отправки команды нода визуально «обрабатывает» запрос (градиент-анимация), потом короткая зелёная вспышка подтверждает применение. Ноды с незаполненными параметрами подсвечиваются пульсирующим оранжевым — их нельзя запустить.

**Состояния ноды:**

- Обычное.
- Выделена (синий контур).
- Требует внимания (оранжевая пульсация).
- Обрабатывает команду (анимация сверху вниз).
- Только что обновилась (зелёная вспышка).
- Готова к запуску (точка в углу).

**Запуск.** Кнопка «Запустить» в шапке прогоняет сценарий через валидацию. Если что-то не так, появляется красная подсказка с конкретной ошибкой:

- «Сигнал не привязан.»
- «У вас есть ноды не готовые к запуску.»
- «Нет пути к ноде Успех.»

Если всё ок — статус кампании меняется на «активна». Кнопка «Сохранить черновик» показывает уведомление «Черновик сохранён» и оставляет кампанию в текущем состоянии.

## Флоу 5. Возврат к работе и управление

### Раздел «Сигналы»

Список карточек. Каждая карточка показывает тип сигнала, количество контактов, дату последнего изменения и две кнопки — «Создать кампанию» и «Скачать». Сортировка — свежие сверху. Одна колонка, без сетки.

**Состояния раздела:**

- **Пусто:** центральное сообщение + две CTA-кнопки («Создать» через мастер / «Загрузить» через файл).
- **Со списком:** карточки по порядку, сверху — кнопка «+ Новый сигнал» с меню.

### Раздел «Кампании»

Список карточек с бейджами статуса. Плоский список, без табов и группировок, сортировка по последнему изменению.

**Четыре статуса кампании:**

- **Не запущено** (черновик) — серый бейдж.
- **Запланировано** — бейдж с датой запуска.
- **Активна** — зелёный бейдж с пульсирующей точкой.
- **Завершена** — приглушённый бейдж.

К статусам добавляется ещё одно состояние — **На паузе** — для активных кампаний, которые временно остановили.

**Состояния раздела:**

- **Пусто:** сообщение «Нет кампаний» и CTA «Создать сигнал →» (потому что без сигнала кампанию не начать).
- **Со списком:** карточки кампаний. Клик по карточке возвращает в canvas этой кампании в том же состоянии, в котором её оставили.

### Раздел «Статистика»

Пункт в меню присутствует, но содержательного наполнения в прототипе пока нет — это будущая точка роста платформы.

## Как связаны разделы

Движение пользователя между разделами всегда направлено к одной цели — **запустить или поддерживать кампанию**.

- **Сигналы** — источник. Без сигнала ничего не происходит.
- **«Запустить»** — самая короткая дорога: либо создать новый сигнал, либо запустить кампанию по существующему.
- **Canvas** — место, где настраивается и запускается конкретная кампания.
- **Кампании** — архив работы: возврат, пауза, возобновление, дублирование существующих сценариев.

Переключение между разделами не теряет состояние — оставленная в середине настройки кампания дождётся возврата ровно там, где её прервали.
