# Campaign Flow — Design Spec

**Date:** 2026-04-04
**Source doc:** `docs/campaign-flow.md`

---

## Scope

Пошаговый wizard создания кампании из 8 шагов. Полностью кликабельный на localhost. Без бэкенда — только UI и состояние в памяти.

---

## Состояние

Всё состояние живёт в `CampaignWorkspace`. При перезагрузке страницы сбрасывается — это намеренно для данного этапа.

```ts
currentStep: number  // 1..8

stepData: {
  scenario: string | null       // шаг 1: id выбранного сценария
  interests: string[]           // шаг 2: выбранные интересы
  triggers: string[]            // шаг 2: выбранные триггеры
  segments: string[]            // шаг 3: выбранные сегменты
  signalLimit: number | null    // шаг 4: лимит сигналов
  file: File | null             // шаг 5: загруженный файл
}
```

Каждый шаг получает `data: StepData` (read-only) и `onNext: (partial: Partial<StepData>) => void`. При вызове `onNext` данные мёржатся и `currentStep` увеличивается на 1.

Навигация назад через Stepper: `setCurrentStep(n)` — данные **не сбрасываются**.

## Анимации

### Смена шагов

**Уход:** текущий контент уезжает вверх за край экрана (`y: 0 → -60, opacity: 1 → 0`).
**Появление:** новый контент появляется на месте, без движения снизу.

Последовательность на новом шаге:
1. Заголовок печатается посимвольно (typewriter)
2. Подзаголовок печатается посимвольно (после заголовка)
3. Карточки / поля / кнопки появляются через fade-in после завершения текста

Реализация: кастомный хук `useTypewriter(text, speed)`. Карточки — `framer-motion` `motion.div` с `initial={{ opacity: 0 }}` и `animate={{ opacity: 1 }}` + `delay` равный суммарному времени печати текстов.

Обёртка в `CampaignWorkspace`:
```tsx
<AnimatePresence mode="wait">
  <motion.div
    key={currentStep}
    exit={{ y: -60, opacity: 0 }}
    transition={{ duration: 0.25 }}
  >
    <CurrentStepComponent ... />
  </motion.div>
</AnimatePresence>
```

### Специальный лоадер — Шаг 5 (хеширование)

Запускается при нажатии "Далее" на шаге 5. Блокирует переход, отображается поверх upload-зоны.

Три последовательных этапа (каждый печатается typewriter-эффектом):
1. «Проверка формата файла...» — ~1.2s
2. «Хеширование данных...» — ~2.0s
3. «Подготовка к импорту...» — ~1.0s

После завершения — автоматический переход на шаг 6.

### Специальный лоадер — Шаг 7 (обработка)

Прогресс-бар фиксированной длины ~4s на весь экран.
Stepper некликабелен. Кнопок нет.
После завершения — автоматический переход на шаг 8.

## Stepper

Появляется начиная с шага 2, в правом верхнем углу рабочей зоны.

| Шаг флоу | Пункт Stepper |
|----------|---------------|
| 2        | Интересы      |
| 3        | Сегменты      |
| 4        | Лимиты        |
| 5        | База          |
| 6        | Сводка        |
| 7        | Обработка     |
| 8        | Результат     |

Состояния пункта: `pending` / `active` / `completed` (галочка).
Клик по `completed` → переход к этому шагу без сброса данных.
Клик по `pending` → заблокирован.
На шаге 7 весь Stepper некликабелен.

---

## Chat Input

Присутствует на каждом шаге, прибит к низу рабочей зоны.
На шаге 1: при выборе карточки подставляется `Сценарий: {name}. ` (уже реализовано).
На остальных шагах: свободный ввод, `console.log` при отправке.

---

## Шаги — сводка условий продолжения

| Шаг | Условие разблокировки кнопки "Продолжить" |
|-----|-------------------------------------------|
| 1   | Клик по карточке → автопереход (кнопки нет) |
| 2   | Выбрана хотя бы одна карточка (интерес или триггер) |
| 3   | Выбран хотя бы один сегмент |
| 4   | Введено целое число > 0 |
| 5   | Файл выбран |
| 6   | Кнопка «Подтвердить и запустить» (всегда активна) |
| 7   | Нет кнопки — автопереход после анимации |
| 8   | Финальный экран, нет перехода дальше |
