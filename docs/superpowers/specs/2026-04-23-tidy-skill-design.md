# `/tidy` Skill — Design Spec

**Date:** 2026-04-23
**Status:** design approved, ready for implementation plan

## Purpose

A manually-invoked Claude Code skill that tidies a project's documentation folder and removes build artifacts. Invoked by the user at natural breakpoints (end of day, end of a roadmap block). Zero auto-detection of "task completion" — the user decides when to run it.

Solves the recurring manual cleanup of `docs/superpowers/plans`, `docs/superpowers/specs`, and gitignored build caches (`.next/`, `tsconfig.tsbuildinfo`, `test-results/`, `.DS_Store`).

## Non-goals (explicitly out of scope for V1)

- Auto-detection of when a plan is "done" — no git heuristics, no frontmatter markers
- Loose markdown files in `docs/` (e.g. `docs/wiki.md`, `docs/2026-04-06-*.md`) — user handles manually
- Wiki sync (validating `wiki.md` against `package.json` / filesystem)
- Roadmap sync (updating memory's `project_afina_*_roadmap.md`)
- Git tagging / version bumping / release changelog
- Touching `node_modules/`, `.worktrees/`, or any other directory not listed in scope

## Invocation

**Command:** `/tidy` (no arguments, no flags).

**Location:** global skill at `~/.claude/skills/tidy/SKILL.md`. Transferable across any project that uses the `docs/superpowers/{plans,specs}/` pattern.

## Flow

```
/tidy
  │
  ▼
1. Precondition check
   ├─ git status must be clean
   └─ if dirty → abort with message "commit or stash first, then re-run /tidy"
  │
  ▼
2. Scan + report
   ├─ list plans in docs/superpowers/plans/ (date, title, days since modified)
   ├─ list specs in docs/superpowers/specs/ (same)
   ├─ list build artifacts present (.next/, tsconfig.tsbuildinfo, test-results/, .DS_Store × N)
   └─ print single-screen summary with sizes
  │
  ▼
3. User reply  (id = plan/spec slug without date prefix, e.g. `campaign-flow`)
   ├─ "archive through YYYY-MM-DD"     → archive plans+specs whose filename date prefix ≤ that
   ├─ "archive: <id1>, <id2>, ..."     → archive specific ids (matches plan slug; pairs spec)
   ├─ "archive all"                    → archive every plan+spec currently in plans/+specs/
   ├─ "skip archive"                    → only clean build artifacts
   └─ "cancel"                          → no changes, exit
  │
  ▼
4. Execute
   ├─ move selected plans → docs/superpowers/archive/plans/
   ├─ move matching specs → docs/superpowers/archive/specs/
   │  (pairing: plan `<date>-<slug>.md` pairs with spec `<date>-<slug>-design.md`
   │   or any spec sharing the same slug — auto-archive the pair together)
   ├─ delete build artifacts
   └─ append entry to docs/superpowers/archive/CHANGELOG.md
  │
  ▼
5. Commit
   └─ single commit: chore(cleanup): archive N plans/specs + clean M MB build cache
```

## Archive layout

Flat structure, preserves plans/specs split:

```
docs/superpowers/
  plans/                         # only active
  specs/                         # only active
  archive/
    plans/
      2026-04-04-campaign-flow.md
      ...
    specs/
      2026-04-04-campaign-flow-design.md
      ...
    CHANGELOG.md
```

Filenames preserved as-is — date prefix stays in the filename for easy chronology. No subfolders by month/block.

## Plan ↔ spec pairing

When user asks to archive a plan, skill auto-archives the matching spec (and vice versa). Pairing rules:

1. For plan `<date>-<slug>.md`, look in `specs/` for:
   - `<date>-<slug>-design.md` (primary match)
   - Any `<date>-<slug>*.md` (e.g. audit variants) — list them to user, ask confirm
2. If no matching spec exists, archive the plan alone. Same in reverse.

## Build artifacts — exact list

The skill removes these if present. All are gitignored; all regenerate automatically.

| Path | Source | Regenerated by |
|---|---|---|
| `.next/` | Next.js build cache | `npm run dev` / `next build` |
| `tsconfig.tsbuildinfo` | TypeScript incremental cache | `tsc` / Next.js build |
| `test-results/` (contents) | Playwright | `playwright test` |
| `**/.DS_Store` | macOS Finder | Finder re-opens folder |

Skill does **not** touch `.next/cache/webpack/` selectively, `node_modules/.cache/`, or any other cache. If the user adds a new build tool later, its artifacts stay until added explicitly to the skill.

## Report format

Single-screen output, grouped by section:

```
📋 Plans (18 files)
  2026-04-04  campaign-flow                     19d ago
  2026-04-05  figma-campaigns-screen            18d ago
  2026-04-05  sidebar-redesign                  18d ago
  2026-04-06  signal-file-attachment            17d ago
  ...
  2026-04-18  workflow-nodes                    5d ago

📋 Specs (20 files)
  [same layout]

🧹 Build artifacts
  .next/                     823 MB
  tsconfig.tsbuildinfo       388 KB
  test-results/              4 KB
  .DS_Store × 2              16 KB
  ─────────────────────────────────
  total                      823 MB

💬 Reply with one of:
  - "archive through YYYY-MM-DD"   archive all plans+specs dated ≤ that
  - "archive: <id1>, <id2>, ..."   archive specific ones (slug or date)
  - "skip archive"                  only clean build artifacts
  - "cancel"                        do nothing
```

Sort order: chronological by filename date prefix (ascending).
"Last modified" is derived from `git log -1 --format=%cr <file>` (last commit touching the file), not filesystem mtime — more accurate across machines.

## CHANGELOG format

`docs/superpowers/archive/CHANGELOG.md` — appended-to, never rewritten:

```markdown
# Cleanup Changelog

Log of `/tidy` runs.

---

## 2026-04-23

**Archived (7 plans, 7 specs):**
- 2026-04-04 campaign-flow
- 2026-04-05 figma-campaigns-screen
- 2026-04-05 sidebar-redesign
- 2026-04-06 signal-file-attachment
- 2026-04-06 ui-polish-cards
- 2026-04-06 ux-navigation-fixes
- 2026-04-06 welcome-page

**Build artifacts cleaned:** 823 MB (.next, tsconfig.tsbuildinfo, test-results, .DS_Store × 2)

**Commit:** `abc1234`

---

## 2026-04-30

[next run...]
```

- Newest entry at the top (below the header).
- Commit SHA added after `git commit` succeeds — skill edits CHANGELOG.md in a follow-up commit, **OR** computes via `git rev-parse HEAD~0` after commit and amends. Pick one at implementation time — see Open questions.

```

Archived:

  plans/ 2026-04-06 signal-file-attachment
  plans/ 2026-04-06 ui-polish-cards
  plans/ 2026-04-06 ux-navigation-fixes
  plans/ 2026-04-06 welcome-page
  specs/ 2026-04-04 campaign-flow-design
  specs/ 2026-04-05 figma-campaigns-screen-design

  .next/ (823MB)
  tsconfig.tsbuildinfo (388KB)

```

Single commit bundles: archive moves + deleted build artifacts + CHANGELOG update.

## Safety rules

1. **Clean tree precondition.** Abort if `git status --porcelain` is non-empty. Message: `working tree is dirty — commit or stash first, then re-run /tidy`. No exceptions, no `--force` flag.
2. **Dry-run by default in V1?** No. Report step (step 2) IS the dry-run — nothing moves until user replies.
3. **No destructive action without an explicit user reply.** "cancel" or timeout / interrupt → skill exits cleanly, nothing moved.
4. **Idempotent.** Running `/tidy` twice in a row: second run shows empty archive candidates + any new build artifacts. No harm.
5. **Archive collisions.** If `docs/superpowers/archive/plans/2026-04-04-campaign-flow.md` already exists (shouldn't happen if tree was clean, but defensively): abort with error, ask user to resolve manually. Never overwrite.

## Open questions (decide at implementation time)

1. **Commit SHA in CHANGELOG**: commit first then amend, or commit CHANGELOG in a separate 2nd commit? Amending is cleaner (one commit total) but slightly harder to implement. Recommendation: amend.
2. **Interactive prompt UX**: does `/tidy` ask its question as plain text in the chat (user replies in natural language), or via the `AskUserQuestion` tool (structured radio buttons)? Recommendation: plain text — more flexible replies like "archive through 2026-04-17 except sidebar-redesign".

## Success criteria

- Running `/tidy` on the current repo (2026-04-23 state) lists all plans + specs in `docs/superpowers/{plans,specs}/` plus any build artifacts in the report.
- Replying `archive through 2026-04-17` moves every plan+spec with filename date prefix ≤ `2026-04-17` into `archive/`, cleans the reported build artifacts, appends a CHANGELOG entry, creates a single commit.
- Running `/tidy` again immediately: reports only plans+specs dated after the archive cutoff, no build artifacts (already cleaned), exits without making changes if user replies `cancel`.
- After a successful run, `git log -1 --format=%s` shows the `chore(cleanup): …` commit and `git status` is clean.
